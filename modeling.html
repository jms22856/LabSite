Modeling

The initial idea behind the "modeling" page was to make it a front-end for my code, i.e. make it an interactive user interface. For example, a user would be able to upload a file and run my code, without having to know the technical details. While I could implement this idea, I'm not sure if TAMU's hosting would allow me to do this. For this reason, I'm probably going to leave any front-end implementations on my personal site. Instead, I'm thinking this page can be used to hold code files, datasets (if we have any we want to share), and similar information.
